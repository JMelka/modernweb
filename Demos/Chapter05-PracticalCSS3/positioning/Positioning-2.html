Absolute Positioning - Basics

Absolute positioning of a child element within a container element. Note that the container is not positioned, i.e. there's no position:xxx set. Therefore the positioned element is positioned relative to the viewport.

There's huge differences in the way that IE and Firefox handle this. Both browsers correctly position the top-left corner of the positioned element box in the centre of the screen (50% - 50%) - try resizing the screen. However:

  • Firefox sizes the positioned element relative to the viewport - the browser window - correctly.
  • IE (all versions) size the positioned element relative to the size of the container element.
  • IE5/Mac gets it wrong on all counts, positioning it vertically to the contaner, horizontally to the viewport, and sizing it to the container.
