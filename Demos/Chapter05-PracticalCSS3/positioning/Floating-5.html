Floating - Creating multi-column layout

Review:

  • you can float left and float right.
  • if two floats meet, they'll 'back up' against each other
  • clearing will move a float downwards until it's against the edge

We'll now extend the floating model and add a third column.

The biggest design criteria is to determine which order the three-column layout should appear in the code.

----------------------------------------------------
| Container                                        |
|  ------------  ------------------  ------------  |
|  | Column A |  |   Column B     |  | Column C |  |
|  |          |  |                |  |          |  |
|  |          |  |                |  |          |  |
|  |          |  |                |  |          |  |
|  |          |  |                |  |          |  |
|  |          |  |                |  |          |  |
|  ------------  ------------------  ------------  |
----------------------------------------------------

Given the layout above, should the HTML appear as Column A, Column B, Column C? This decision is based upon your requirements for non-CSS browsers (mobile phones, PDAs, screen readers, etc) and not forgetting search engines.

Firstly, we need to draw our structure to understand how the sections fit together. This is a really important step; developing stylesheets can get rather messy and it's easy to get confused with the the big picture. It is recommended that you place a drawing like this into the actual stylesheet - you will appreciate it when you make changes in a year's time! 

---------------------------------------------------------------------------------
|  div = CentreSection   (centred in the page)                                  |
|  -----------------   -------------------------------------------------------  |
|  |               |   | div = RightNavAndContent  (right float)             |  |
|  |               |   |   ----------------------------   -----------------  |  |
|  | div = LeftNav |   |   | div = Content            |  | div = RightNav |  |  |
|  | (left float)  |   |   | (left float)             |  | (right float)  |  |  |
|  |               |   |   |                          |  |                |  |  |
|  |               |   |   |                          |  |                |  |  |
|  |               |   |   |                          |  ------------------  |  |
|  -----------------   |   |                          |                      |  |
|                      |   |                          |                      |  |
|                      |   |                          |                      |  |
|                      |   |                          |                      |  |
|                      |   ----------------------------                      |  |
|                      |                                                     |  |
|                      -------------------------------------------------------  |
|                                                                               |
---------------------------------------------------------------------------------

The problem with this layout is the order of the columns in the HTML: first Content, second RightNav, thirdly LeftNav. This isn't normaly what you want (in a left-to-right language). You normally want the order to be Content/LeftNav/RightNav. So we'll re-design this: 

--------------------------------------------------------------------------------------
|  div = CentreSection   (centred in the page)             600px wide                |
|  -----------------------------------------------------------   ------------------  |
|  | div = LeftNavAndContent (LEFT float) (600-140-10=450px) |   | div = RightNav |  |                   
|  |  -----------------   ---------------------------------  |   |                |  |
|  |  | div = LeftNav |   | div = Content                 |  |   |                |  |
|  |  | 150px         |10 |                               |  |10 | 140px          |  |
|  |  |               |px |  width=600-140-10-10-140      |  |px |                |  |
|  |  |               |   |       =600-300                |  |   ------------------  |
|  |  |               |   |       =300px                  |  |                       |
|  |  -----------------   |                               |  |                       |
|  |                      |                               |  |                       |
|  |                      |                               |  |                       |
|  |                      |                               |  |                       |
|  |                      |                               |  |                       |
|  |                      ---------------------------------  |                       |
|  -----------------------------------------------------------                       |
|                                                                                    |
--------------------------------------------------------------------------------------
The affect of different source order on layout
Column Source Code Order Floating Strategy
A, B, C Float A and B left, float C right
B, A, C Foat B left, and float C right in a container
Float this container right
Float A left
C, B, A Float C right, float B left both in a container
Float this container right
Float A left

Now to the HTML. Our strategy is the more common layout with the Content area to appear at the top of the HTML (good for non-CSS browsers and very good for search engines), the LeftNav second, and the RightNav last.

Now design the stylesheet

#CentreSection   { width:600px; 
                   margin:0 auto; /* centre the container on the page */
                   border-color:#369; border-width:1px 10px; border-style:solid; 
                   padding:0; /* adjust padding using sub-containers */
                   background:#f99; /* temporary background colour */}
#LeftNavAndContent { width: 450px; 
                   float:left;
				   margin:0; border:0; padding:0;
				   background:#f9f; /* temp */ }
#LeftNav {         float:left;
                   width:140px;
                   margin:0;
                   border:1px solid black;; /* temp border */
                   background:#9f9; /* temporary background colour */}
#Content {         float:right;
                   width:430px; /* 600-160=440, plus add a gutter */
                   margin:0; border:0; padding:0;
                   background:#ff9; /* temporary background colour */}
#RightNav {        float:right;
                   width:140px;
                   margin:0;
                   border:1px solid black;; /* temp border */
                   background:#99f; /* temporary background colour */}
#LeftNav ul,
#RightNav ul {     word-wrap:break-word; /* force IE to wrap long text in the menus */ }

And add the stylesheet code. This has basically been copied from Position-13.html and been tweaked for colour.

The one major change is suppressing the last item's border. We set an id on the last item "FooterLastItem" which was styled with a border-right:0 rule. We used an ID as you may want to use the class for other things such as the crumbtrail. 

#FooterNav { clear:both; /* ensure the CentreSection container encloses the floats */   
                  background:#369; /* same colour as the border of the CentreSection */   
                  margin:0;   
                  border-top:#369 solid 2px;   
                  padding:0;   
                  color:#fff;   
                  list-style:none;}   
#FooterNav ul {   margin:0; border:0; padding:0; /* reset the margins, etc. */   
                  text-align:center;  /* as the list items are displayed inline, they can be centred using text-align */   
                  background:#369; }   
#FooterNav li {   display:inline;   
                  line-height:2em;   
                  margin:0;   
                  padding:0 0.5em;   
                  border-top:0; border-right:1px solid #999; border-bottom:0; border-left:0; }   
#FooterNav li a { text-decoration:none;   
                  color:#ddd;   
                  margin:0;   
                  border:0;   
                  padding:0.2em 0.5em; }   
#FooterNav li a:hover {  text-decoration:underline;   
                  background-color:#fff;    
                  color:#339; }   
#FooterNav .CrumbTrail { background-color:#fff; color:#339; }   
#FooterNav li#FooterLastItem { border-right:0; /* border for the first footer menu item */ }
