Positioning - relative positioning

When a child is positioned relative to its parent, it is offset from the 'hole' it would nomally occupy. The 'hole' is not closed up. The element will overwrite whatever gets in its way.

Most browsers support relative positioning. 

<div style="width:400px; height:400px; 
            margin:0; border:5px solid #369; padding:0; background:#ffc;">
      <h1>Container Element</h1>
      <div style="float:left; position:relative; width:100px; height:100px; 
                  background:#fcf; left:100px; top:100px; 
                  margin:0; border:5px solid #f00; padding:0;">
            Positioned element
      </div>
      <p>Lorum ipsum dolor sit amet........
            sm dolor sit amet.</p>
</div>

Positioning - relative positioning

Setting a negative offset moves the element outside of the containing block. The browser calculates the scrollbars according to the position of the 'hole'. Thus it's quite possible to render an element below or above the screen. 

<div style="width:400px; height:400px; 
            margin:0; border:5px solid #369; padding:0; background:#ffc;">
      <h1>Container Element</h1>
      <div style="float:left; position:relative; width:100px; height:100px; 
                  background:#fcf; left:-50px; top:-50px; 
                  margin:0; border:5px solid #f00; padding:0;">
            Positioned element
      </div>
      <p>Lorum ipsum dolor sit amet.........
            sm dolor sit amet.</p>
</div>
