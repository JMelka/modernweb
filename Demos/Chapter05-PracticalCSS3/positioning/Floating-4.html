The second trick harks back to the bad old days of tables and spacer cells. Nothing to be proud of! It's known as "Faux Columns"

Create a 1px high but wide graphic in the colour of the background of the menu, then add it to the background of the CentreSection. We've created a tiny 1 px high, but 800px wide graphic, LeftNavMenuBackground.gif. We can use this along with background attachment to set the colour on the left column.

Notice that we've used a negative X attachment to shove the image to the left and leave exactly 160px showing. Sure, you could create an exact distance, but this technique will work with relative sizing (em) and means you don't need to change the graphic if you want a wider menu area. You could also have a grahic that shows a gradient or beveled edge.

There is one major issue with this technique - apart from it being an obvious hack - which is related to how browsers handle images. It is possible that a browser will use a different 'gamut' (see http://en.wikipedia.org/wiki/Gamut) and there will be very subtle differences between the background colour and the image colour. One interesting one the author's seen is black - it was different between a browser's 'black' and the image's black!

Now the only thing left to do is reset the colours to the designer's vision. Set the background colour of the CentreSection to the same colour of the Content (normally white), and the background colour of the LeftNav to match the graphic (light grey).

For more info on another way around this issue, see http://positioniseverything.net/articles/onetruelayout/equalheight. 

<div id="CentreSection" style="background:red url(LeftNavMenuBackground-Cyan.gif) 0 -540px repeat-y;">
