Box Content - p

 

Interesting things happen when you don't specify the height of the containing block. We've added the rule height:auto to reset the height.

  • Internet Explorer displays it pretty much the same as above - adjusting the height of the container to fit the floated box.
  • Firefox removes the float from it's Container height calculation, and as the container has no other contents, collapses the container leaving the box outside. The boxes only butt against each other as the padding & margins are set to 1em.
    This can be fixed in a couple of ways:
    • Add a height to the container (as we've done above)
    • Add another element below the <p> with a style of clear:both (strictly speaking we only need a clear:right) 
<div id="ContainersContainer">
    <div id="Container" style="height:auto">
        <p>Box Content - p</p>
    </div>
</div>

Now using an image that's too big for the container. This highlights browser differences:

  • Internet Explorer - expands the container to fit the image but does not reflow the text which is still limited to 400px.
  • Firefox - doesn't attempt to expand the container, instead displays the image off the left of the screen and truncated.
