Floating - Two column layout using floats

Now to start using floats. We're going to create the centre section of the website, between the header and footer. The design calls for a left-navigation menu and the page contents which both are of an unknown height.

Firstly, we need to draw our structure to understand how the sections fit together. 

-------------------------------------------------------
|  div = CentreSection   (centred in the page)        |
|  -----------------    ----------------------------  |
|  | div = LeftNav |    | div = Content            |  |
|  | (left float)  |    | (right float)            |  |
|  |               |    |                          |  |
|  |               |    |                          |  |
|  |               |    |                          |  |
|  |               |    |                          |  |
|  -----------------    |                          |  |
|                       |                          |  |
|                       |                          |  |
|                       |                          |  |
|                       ----------------------------  |
-------------------------------------------------------

From this drawing, it is clear that we need to define three sections in our document. We will also need to ensure there's a clear at the bottom to force the centre section down. Our basic HTML will be: 

<div id="CentreSection">
   <div id="LeftNav">
      <ul>
         <li>Menu 1</li>
         <li>Menu 2</li>
         <li>Menu 3</li>
         <li>Menu 4</li>
      </ul>
   </div>
   <div id="Content">
      <h1>Welcome to the content</h1>
      <p>Lorum ipsum...</p>
   </div>
</div>

There is a small issue with this in that it would be much nicer to have the content appear before the menu in the source code. This is better for people using non-CSS aware devices (mobile phone, PDA, screen reader, etc.), and it is much better for web search engines. As we're using floats, this is not a problem.

<div id="CentreSection">
   <div id="Content">
      <h1>Welcome to the content</h1>
      <p>Lorum ipsum...</p>
   </div>
   <div id="LeftNav">
      <ul>
         <li>Menu 1</li>
         <li>Menu 2</li>
         <li>Menu 3</li>
         <li>Menu 4</li>
      </ul>
   </div>
</div>

Now to get on with some styling. Firstly create the basic styles with widths and floats: 

#CentreSection { width:600px; 
                 margin:0 auto; /* centre on the page */
                 border-color:#369; border-width:1px 10px; border-style:solid; 
                 padding:0;
                 background:red; /* temporary background colour */}
#LeftNav {       float:left;
                 width:160px;
                 margin:0;
                 border:1;
                 background:green; /* temporary background colour */}
#Content {       float:right;
                 width:430px; /* 600-160=440, plus add a gutter */
                 margin:0; border:0; padding:0;
                 background:yellow; /* temporary background colour */}

Where has the menu gone in IE? It's extended the width of the LeftNav float such that there's no space for it to fit in the gap between the Content float and the side. So it then moves the float downwards until if finds space.

We're fortunate that the workaround is to simply add a Microsoft proprietary CSS rule, word-wrap:break-word;

The down side to using this is that your stylesheet won't validate. If this is imporant to you, you can move this rule to a separate stylesheet and hide them using conditional comments. 

       <ul style="word-wrap:break-word;">
            <li>Llanfairpwllgwyngyllgogerychwyrndrobwllllantysiliogogogoch is a village in Wales</li>
       </ul>
